路由的可解释性与稳定性

  • 黑箱问题： 为什么某个 token 被路由到特定专家？当前没有明确的可解释性

  • 路由坍缩： 训练不稳定时，少数专家主导全部流量——辅助损失不能完全解决

  • 开放问题： 能否设计具有 可解释语义的路由？ （如显式按领域、语法类别路由）

  • 相关工作：[Zuo et al., 2022] Taming Sparsely Activated Transformer

超长序列下的负载均衡

  • 现有辅助损失假设 batch 内 token 分布均匀 → 长序列时局部 token 高度相关，假设失效

  • 同一文档的 token 可能高度相似 → 大量 token 涌向同一专家 → 局部 collapse

  • 开放问题： 序列感知（sequence-aware）的路由与均衡策略

  • 潜在方向： per-sequence 辅助损失、动态容量分配

MoE 与长上下文的协同优化

  • 绝大多数 MoE 模型在短上下文上预训练， 长上下文是微调而非预训练——专家分工可能不适用

  • 路由模式在长序列和短序列上可能 截然不同——短文本专家分工在长文本上未必有效

  • 开放问题： 如何在长上下文预训练中维持高效的 MoE 路由？

  • 工程挑战： Ring Attention + Expert Parallelism 的集成复杂度极高
