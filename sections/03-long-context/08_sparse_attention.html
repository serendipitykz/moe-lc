稀疏注意力是"用近似换效率"的路线，FlashAttention 是"精确但 IO 高效"的路线。两条路线在 2022 年后基本收敛为 FlashAttention 主导。Longformer 和 BigBird 的意义在于证明了理论可行性，Sliding Window 依然在 Mistral 中使用。

稀疏注意力模式

通过限制注意力范围将 \(O(L^2)\) 降为 \(O(L \cdot w)\)——精确度换效率

Longformer [Beltagy et al., 2020]

局部窗口 + 全局 token

每个 token 仅注意 ±w/2 个相邻 token（局部）；特殊全局 token（如 [CLS]）双向注意所有位置。

\[O(L \cdot w) + O(L \cdot n_g)\]

w = 窗口宽，\(n_g\) = 全局 token 数

BigBird [Zaheer et al., 2020]

局部窗口 + 全局 token + 随机注意力

在 Longformer 基础上增加每行若干随机注意力点，覆盖长程依赖。理论证明：该组合在 O(L) 注意力下图灵完备。

\[O\!\left(L(w + n_g + r)\right) = O(L)\]

r = 每行随机注意力点数（通常 r ≈ 3）

Sliding Window

Mistral 7B [Jiang et al., 2023]

每 token 仅注意最近 w = 4096 个 token，无全局 token。结合 GQA（8 组）和 RoPE，实现 32K 上下文。

\[O(L \cdot w),\quad w = 4096\]

稀疏 vs 精确注意力

稀疏注意力：近似，牺牲长程依赖；工程复杂，需自定义 CUDA kernel。

FlashAttention：精确，仅减少 IO；对模型行为无影响，已成现代标配。

稀疏模式在 2022 年后逐渐被 FlashAttention 取代，但 Sliding Window 仍见于 Mistral/Gemma 等高效模型。
