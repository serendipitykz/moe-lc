NSA 和 MoBA 是 2025 年稀疏注意力的新范式：不再手工设计稀疏规则，而是让模型端到端学习"哪里值得精确关注"。NSA 三路分支各司其职；MoBA 借鉴 MoE 的 top-k 门控。两者分别代表 token 级和 block 级的动态稀疏。 --- 我们快速回顾一下 2020 年那批稀疏注意力工作的共同局限：Longformer、BigBird、Sliding Window 都是手工设计稀疏规则——不管输入是什么，注意力模式是固定的。2022 年 FlashAttention 出现后，精确注意力的 IO 问题得到了解决，静态稀疏注意力基本被淘汰。 但到了 2025 年，稀疏注意力以完全不同的面貌回归了——这次是**动态的、可学习的**。 **NSA（Native Sparse Attention）**，DeepSeek-AI + PKU，2025 年 2 月。 NSA 的核心是三路并行分支，最后通过可学习门控 g_t^c 融合输出： 第一支路 Compression：把输入序列按 l=32 token 为一组，用 MLP 把每组压缩成单个 token 后做注意力。粗粒度全局摘要。 第二支路 Selection：在 block 级别评分，选 top-16 块（每块 64 token），做精细注意力。精确检索最重要区域。 第三支路 Sliding Window：窗口 w=512，保留局部近邻依赖。 三路加权融合：o_t* = Σ_c g_t^c · Attn(q_t, K_t^c, V_t^c)。GQA 兼容，Triton kernel 实现，可端到端训练，无需人工调参。 实测：64K 上下文下 9 倍 prefill 加速、11.6 倍解码加速；7/9 通用任务优于全注意力基线。 **MoBA（Mixture of Block Attention）**，Moonshot AI + 清华大学，2025 年 2 月。 MoBA 的设计直接借鉴了 MoE 的 top-k 路由思想：把长序列划分为等大块，用当前 query 对每块的 key 做均值池化后打分，选 top-k 块做精细注意力，同时始终保留当前块（完全对应 shared expert 的设计）。 门控公式：s_i = q · mean_pool(K[block_i])。这个操作非常轻量，不需要额外的参数。 在 32K 序列上，MoBA 的稀疏度达到 95.31%；128K 上的 RULER benchmark 得分 0.782，而全注意力是 0.785，差距极小。1M 上下文 prefill 加速 6.5 倍。训练时用 90% MoBA + 10% 全注意力混合，保留全局理解能力。 这两个工作的出现，把稀疏注意力的范式从"静态手工设计"推进到了"动态端到端学习"。这是 2025 年长上下文领域最重要的趋势之一。
