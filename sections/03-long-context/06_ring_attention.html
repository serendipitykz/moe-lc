Ring Attention 是长上下文的"分布式解法"。类比：一个大圆桌上 P 个人互相传菜单（K/V），每个人同时对自己面前的菜单做计算。通信和计算完全重叠，所以没有延迟代价。 ---------- 如果序列长到连 FlashAttention + GQA 都撑不住怎么办？答案是 **Ring Attention** [Liu et al., 2023]。 思路很简单：把序列切分给多张 GPU。每张 GPU 持有 L/P 的 token（Q、K、V 都切分），然后按"环形"顺序依次把 K/V 块传递给邻居，每步在本地做 FlashAttention 并把结果累积进来。经过 P 步之后，每张 GPU 都积累到了完整的注意力输出，等价于全局注意力。 关键优势：通信（发送 K/V 块到下一张 GPU）和计算（对本地 K/V 块做 FlashAttention）可以**完全重叠**。通信几乎被"藏"在计算里，接近线性扩展。 显存复杂度从 O(L) 降到 O(L/P)，计算从 O(L²) 降到 O(L²/P)。这使得 1M token 的序列在足够多 GPU 上成为可能。 Ring Attention 和 Expert Parallelism 结合时，工程复杂度极高——两种通信模式需要精心调度，这是当前超长上下文 MoE 训练的核心挑战之一。

Ring Attention 与序列并行

将超长序列分布到多个 GPU，打破单卡显存上限

问题：单 GPU 无法容纳超长序列

FlashAttention 将 KV Cache 空间降至 \(\mathcal{O}(L)\)，但当 \(L = 1\text{M}\) 时 KV Cache 仍需 数百 GB，远超单卡显存。

Ring Attention 核心思想 [Liu et al., 2023]

  • 将序列切分为 \(P\) 块，每块分配一张 GPU：\(L/P\) tokens/GPU
  • 每张 GPU 持有本块的 Q、K、V 分片
  • 按"环形"顺序逐步传递 K/V 块，每步在本地执行 FlashAttention
  • 经 \(P\) 步后，每个 GPU 累积到完整的注意力输出

关键优势：通信-计算重叠

GPU 在执行本地 FlashAttention 的同时将下一块 K/V 发送给邻居， 通信几乎被掩盖，端到端吞吐接近线性扩展。

显存：\(\mathcal{O}(L/P)\) per GPU
计算：\(\mathcal{O}(L^2 / P)\) per GPU
