Ring Attention 与序列并行

将超长序列分布到多个 GPU，打破单卡显存上限

问题：单 GPU 无法容纳超长序列

FlashAttention 将 KV Cache 空间降至 \(\mathcal{O}(L)\)，但当 \(L = 1\text{M}\) 时 KV Cache 仍需 数百 GB，远超单卡显存。

Ring Attention 核心思想 [Liu et al., 2023]

  • 将序列切分为 \(P\) 块，每块分配一张 GPU：\(L/P\) tokens/GPU
  • 每张 GPU 持有本块的 Q、K、V 分片
  • 按"环形"顺序逐步传递 K/V 块，每步在本地执行 FlashAttention
  • 经 \(P\) 步后，每个 GPU 累积到完整的注意力输出

关键优势：通信-计算重叠

GPU 在执行本地 FlashAttention 的同时将下一块 K/V 发送给邻居， 通信几乎被掩盖，端到端吞吐接近线性扩展。

显存：\(\mathcal{O}(L/P)\) per GPU
计算：\(\mathcal{O}(L^2 / P)\) per GPU
