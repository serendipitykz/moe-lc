线性注意力把注意力复杂度降到 O(n)，但表达力有损失。混合架构（如 MiniMax-01）用少量 Softmax 层补足长程推理能力，是当前实用化的最优路线。 --- 前面我们讲的方案——稀疏注意力、FlashAttention——都在 O(L²) 框架内优化，要么减少访问量，要么减少 IO。能不能直接把注意力本身变成 O(L) 的？这就是**线性注意力**的思路。 **线性注意力的核心：φ(Q)·(φ(K)^T V)** 标准 Softmax 注意力是 softmax(QK^T)V，复杂度 O(n²d)，因为 n×n 的注意力矩阵必须完整计算。线性注意力的思路是：用一个特征映射函数 φ，把 Q 和 K 都映射到正数域，然后改变运算顺序： 原来：(QK^T)V = n×n 矩阵乘 n×d → O(n²d) 变为：Q(K^T V) = n×d 矩阵乘 d×d → O(nd²) 当 d 远小于 n 时，这是 O(n) 的。这个等价性来自矩阵乘法的结合律——先算 K^T V（一个 d×d 的小矩阵），再左乘 Q。 更进一步，线性注意力可以用一个递推形式：维护一个历史累积器 context_t = Σ_{i=1}^{t} φ(k_i)v_i^T，每来一个新 token 只需更新这个 d×d 的矩阵，而不需要访问所有历史 KV。这使得它可以像 RNN 一样高效地做流式推理。Flash-Linear Attention 库为各种线性注意力变体提供了融合 CUDA/Triton 内核，速度与 FlashAttention 相当。 **线性注意力的局限** 但线性注意力的代价也是真实的。φ 函数是对 softmax 的近似，会丢失选择性——模型无法精确聚焦在某个特定位置，而是把所有历史信息平均混合。在需要精确检索的任务（比如 NIAH）上，纯线性注意力的表现明显差于 Softmax。 **混合架构：Lightning Attention + Softmax（MiniMax-01）** MiniMax-01 的解决方案是混合：在 80 层的模型里，每 8 层一个周期，7 层用 Lightning Attention（线性注意力），1 层用标准 Softmax Attention。比例是 7:1。 Lightning Attention 的实现也有一个技巧：把序列切成 chunk，chunk 内部用标准的因果注意力（O(chunk_size²)），chunk 之间用线性注意力的递推形式（O(n·d)，无因果掩码）。这样既保留了局部精度，又实现了全局 O(n) 的复杂度。RoPE 位置编码只加在 Softmax 层，不加在 Lightning 层（因为线性注意力不需要精确的位置感知）。 结果：在长上下文的"big needle"任务上，7:1 混合架构的准确率接近全 Softmax 模型，但计算复杂度大幅下降，支持百万级上下文。 **为什么 7:1 而不是全用线性？** 核心原因是**稀疏检索能力**。有些任务需要从百万 token 里精确召回特定信息，这需要 softmax 的尖锐注意力分布。7 层线性注意力负责积累"背景知识"，1 层 Softmax 负责精确检索，分工明确。这是工程实践中的最优权衡点。

线性注意力与混合架构

将 O(L²) 降至 O(L)——历史累积器形式 · 混合 Softmax 补足精确检索能力

线性注意力

核心：用特征映射 \(\phi\) 替代 Softmax 核

\[\operatorname{softmax}(QK^\top)V \;\xrightarrow{\;\phi\;}\; \phi(Q)\,\bigl(\phi(K)^\top V\bigr)\]

\(\phi\)（如 ELU+1）将 Q/K 映射到非负空间，近似 Softmax 核，使括号调换合法。

改变运算顺序，\(O(n^2 d)\) → \(O(n d^2)\)：

\[\underbrace{(\phi(Q)\phi(K)^{\top})V}_{O(n^2 d)} \;\longrightarrow\; \underbrace{\phi(Q)\,(\phi(K)^{\top}V)}_{O(nd^2)}\]

等价递推（可流式推理）：

\[\mathbf{C}_t = \mathbf{C}_{t-1} + \phi(k_t)\,v_t^{\top}, \quad \text{out}_t = \phi(q_t)\,\mathbf{C}_t\]

\(\mathbf{C}_t \in \mathbb{R}^{d\times d}\)——历史积累器，替代完整 KV Cache；Flash-Linear Attention 提供融合 kernel。

局限：近似精度损失

φ 函数无法重现 Softmax 的"尖峰"注意力——精确 NIAH 检索性能下降明显。需与 Softmax 层混合使用。

混合 Lightning Attention [MiniMax-01, 2025]

Lightning Attention：chunk 内部因果 Softmax（局部精度，\(O(\text{chunk}^2)\)）＋ chunk 间线性递推（全局 \(O(n)\)），RoPE 仅加在 Softmax 层。80 层模型中每 8 层一周期：7 层 Lightning + 1 层 Softmax（比例 7:1）。

RoPE 位置编码仅施加于 Softmax 层（线性注意力不需要精确位置感知）。

设计哲学

7 Lightning 层 积累背景知识（O(n) 高效） 1 Softmax 层 精确检索关键信息

实测效果

long-context "big needle" 任务准确率接近全 Softmax 模型，支持百万级上下文，计算复杂度大幅低于全 Softmax。
