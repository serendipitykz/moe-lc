KV Cache 与高效注意力变体

KV Cache

推理时对每个 token 缓存 Key 和 Value 向量，避免对历史 token 重复计算注意力。

每层 KV Cache 大小（FP16）：

\[2 \times L \times H \times d_{\text{head}} \;\text{字节}\]

以 Llama-3-70B，\(L = 131\text{K}\) 为例：[Meta AI, 2024]

\(2 \times 131072 \times 64 \times 128 \times 2 \approx\) ~4.3 GB / 层

× 80 层 = ~344 GB 仅用于 KV Cache！

KV Cache 随 \(L\) 线性增长——长上下文下的核心显存瓶颈。

PagedAttention (vLLM)[Kwon et al., 2023]

借鉴操作系统虚拟内存分页机制管理 KV Cache，支持动态 batching，大幅提升推理吞吐量。
