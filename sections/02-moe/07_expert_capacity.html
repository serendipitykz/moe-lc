Token dropping 是 MoE 训练中被低估的实际问题。用"餐厅桌位有限，超出容量的顾客被跳过"来类比。

专家容量与 Token 丢弃

Expert Capacity

\[ C = \left\lceil \frac{\text{tokens per batch}}{N_{\text{exp}}} \right\rceil \times \text{capacity factor} \]

capacity factor ≈ 1.0–1.25（训练），≈ 2.0（推理） [Fedus et al., 2022]

影响

Token dropping 导致信息损失，长序列时尤为严重

缓解方案

增大 capacity factor（增加 padding 开销）/ Expert Choice 路由（专家主动选 token）
