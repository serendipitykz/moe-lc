核心机制。用"考试选题"类比：每道题（token）从题库（专家）中选最相关的 k 道来解答。强调门控网络 W_g 本身很小。 ---------- 路由机制的细节来了。 门控函数 g(x) = TopK(Softmax(W_g · x), k)。W_g 是一个 d_model × N_exp 的矩阵，把输入 token x 映射到一个 N_exp 维的分数向量，过 Softmax 归一化后，取分数最高的 k 个专家。 整层的输出是：y = Σ_{i ∈ T_k} g_i(x) · FFN_i(x)，其中 T_k 是被选中的 k 个专家的下标集合。每个专家输出乘以对应的门控权重，再求和。 这里有一个重要的工程细节需要注意：**非激活专家在反向传播中也不更新梯度**。这意味着，如果某个专家一直得不到路由，它的权重就永远得不到更新，变成"僵尸专家"。这是 MoE 训练需要特别处理的问题，我们接下来马上讲。

Top-K 路由机制

Gating Function

\[ \mathbf{g}(x) = \text{TopK}\!\left(\text{Softmax}(W_g \cdot x),\; k\right) \]

Layer Output

\[ y = \sum_{i \in \mathcal{T}_k} g_i(x) \cdot \text{FFN}_i(x) \]

其中 \(\mathcal{T}_k\) 为 top-k 选中的专家下标集合

每个 token 独立计算门控分数，仅激活得分最高的 k 个专家，其余专家参数不参与当前 token 的前向计算。[Shazeer et al., 2017]

⚠ 非激活专家的梯度：在反向传播中也不更新，可能导致专家分化不均
