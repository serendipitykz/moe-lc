规模定律：越大越好？

Scaling Laws
  • Kaplan et al. (2020)：性能随参数量 N、数据量 D、算力 C 呈幂律增长
  • Chinchilla (2022)：算力最优训练中，N 与 D 应同比例增长
  • GPT-3：1,750 亿参数  →  Llama 3：4,050 亿参数  →  还要更大？
核心矛盾：参数量 10×  ⇒  推理成本 10×。模型训练一次，但推理成千上万次。
能否让模型「拥有」更多参数，却只「使用」其中一部分？

稠密 vs 稀疏激活

Dense vs Sparse Activation

Dense（稠密）

  • 每个 token 激活全部参数
  • 计算量正比于总参数量
  • 代表模型：GPT-4、Llama 3
FLOPs ≈ 2 · N · T

Sparse MoE

  • 每个 token 只激活少数专家
  • 计算量正比于激活参数量
  • 代表模型：Mixtral、DeepSeek-V2
FLOPs ≈ 2 · Nact · T   (Nact ≪ N)
稀疏激活：以 Dense 级别的推理开销，撬动更大的模型容量

MoE 核心机制

How MoE Works
  • 将 Transformer 各层的 FFN 替换为 E 个并行的专家网络
  • 门控网络（Router）为每个 token 计算专家得分，选取 top-k 个专家
  • 仅被选中的 k 个专家参与前向计算，其余保持静默
  • 各专家输出按路由权重加权求和，得到该层最终输出
y = ∑i ∈ TopK(x)   Gi(x) · Experti(x)
Mixtral 8×7B：总参数 ~46.7B，每 token 激活 ~12.9B（≈ 7B dense 的推理成本）
